          On the 2nd I had IPTV installed at my house.  I wanted to use my ERL3 but it didn't work.  For the short term I decided to use their modem, but my ideal situation is to use the ERL3.

           

          ETH0 - my network

          ETH1 - Wan

           

          I had configured my ERL3 with the Wan2lan configuration.

           

          Here is a picture of their router configuration... http://snag.gy/YMM6u.jpg

           

          I went through the IGMP Cli commands to attempt to setup an igmp-proxy, but didn't know what else to try, so I came here.

           

          Here are some assumptions:

          1. Their IPTV uses IGMP v3.

          - Saw this article. http://forums.smallnetbuilder.com/archive/index.php?t-13949.html

          - I called them today and spoke with the rep, he said their IPTV works via multicast.

          2. I'm missing something obvious.

           

           

           

           

          So I did some more testing and some things do not make sense

           

          1. I called tech support to find out if they have any vlans for their video

          - Tech told me that they just use multicast for the boxes to communicate.

           

          2. If I press "Guide", in the preview box, I can see the TV channel only when I change the channel.  If I select the channel, I get ~5 seconds of TV before it freezes and says 'video lost'

           

           

          My problem:

          - When I use my ERL3, my IPTV does not work.

           

          Solution:

          Use cruddy Actiontec router (bad solution).

           

           

          I'm using the ERL3 v1.6

          Service Provider for the IPTV is TDS Telecom.

          Here is the config, I removed some things like static mappings and firewall stuff..

           

          Also do note that you'll need to setup the igmp-proxy:  http://wiki.ubnt.com/IGMP_-_CLI_Commands

           

          If for some reason your tv isn't working with the current config. 

          What I did was look at the multicast via this command:  show ip multicast mfc

          and make sure the origin was  proxied 

           

           

          firewall {
    all-ping enable
    broadcast-ping disable
    group {
    }
    ipv6-receive-redirects disable
    ipv6-src-route disable
    ip-src-route disable
    log-martians enable
    name WAN_IN {
        default-action drop
        description "WAN to internal"
        rule 2 {
            action accept
            description "Allow established/related"
            state {
                established enable
                related enable
            }
        }
        rule 3 {
            action accept
            description "Allow Multicast"
            destination {
                address 239.0.0.0/8
            }
            log disable
            protocol all
        }
        rule 4 {
            action drop
            description "Drop invalid state"
            state {
                invalid enable
            }
        }
        rule 5 {
            action accept
            description "Allow UDP to Multicast"
            destination {
                address 239.0.0.0/8
            }
            log disable
            protocol udp
            state {
                established disable
                invalid disable
                new enable
                related disable
            }
        }
    }
    name WAN_LOCAL {
        default-action drop
        description "WAN to router"
        rule 2 {
            action accept
            description "Allow established/related"
            state {
                established enable
                related enable
            }
        }
        rule 3 {
            action accept
            description "Allow Multicast"
            destination {
                address 239.0.0.0/8
            }
            log disable
            protocol all
        }
        rule 4 {
            action accept
            description "Allow UDP to Multicast"
            destination {
                address 239.0.0.0/8
            }
            log disable
            protocol udp
            state {
                established disable
                invalid disable
                new enable
                related disable
            }
        }
        rule 5 {
            action accept
            description "Allow IGMP"
            log disable
            protocol igmp
        }
        rule 6 {
            action drop
            description "Drop invalid state"
            state {
                invalid enable
            }
        }
    }
    receive-redirects disable
    send-redirects enable
    source-validation disable
    syn-cookies enable
}
interfaces {
    ethernet eth0 {
        address 10.0.0.1/24
        description "Local Intranet"
        duplex auto
        speed auto
    }
    ethernet eth1 {
        address dhcp
        description "TDS Internet"
        duplex auto
        firewall {
            in {
                name WAN_IN
            }
            local {
                name WAN_LOCAL
            }
        }
        speed auto
    }
    ethernet eth2 {
        address 192.168.2.1/24
        description Ooma
        duplex auto
        speed auto
    }
    loopback lo {
    }
}
port-forward {
    auto-firewall enable
    hairpin-nat enable
    lan-interface eth0
    rule 1 {
        description DeleteThisToo
        forward-to {
            address 10.0.0.2
            port 2222
        }
        original-port 2222
        protocol tcp_udp
    }
    rule 2 {
        description deleteThis
        forward-to {
            address 10.0.0.1
            port 1111
        }
        original-port 1111
        protocol tcp_udp
    }
    wan-interface eth1
}
protocols {
    igmp-proxy {
        interface eth0 {
            alt-subnet 10.0.0.1/24
            role downstream
            threshold 1
        }
        interface eth1 {
            alt-subnet 184.61.0.0/16
            alt-subnet 184.60.0.0/16
            alt-subnet 224.0.0.0/4
            role upstream
            threshold 1
        }
    }
}
service {
    dhcp-server {
        disabled false
        hostfile-update disable
        shared-network-name Intranet {
            authoritative disable
            subnet 10.0.0.0/24 {
                default-router 10.0.0.1
                dns-server 10.0.0.51
                lease 86400
                start 10.0.0.100 {
                    stop 10.0.0.200
                }
                unifi-controller 10.0.0.50
            }
        }
        shared-network-name LAN2 {
            authoritative disable
            subnet 192.168.2.0/24 {
                default-router 192.168.2.1
                dns-server 192.168.2.1
                lease 86400
                start 192.168.2.21 {
                    stop 192.168.2.240
                }
            }
        }
    }
    dns {
        forwarding {
            cache-size 150
            listen-on eth0
            listen-on eth2
        }
    }
    gui {
        https-port 443
    }
    nat {
        rule 5010 {
            outbound-interface eth1
            type masquerade
        }
    }
    ssh {
        port 22
        protocol-version v2
    }
}
system {
    host-name ubnt
    login {
        user nouser {
            authentication {
                encrypted-password ""
                plaintext-password ""
            }
            level admin
        }
    }
    name-server 216.165.129.158
    name-server 216.170.153.146
    ntp {
        server 0.ubnt.pool.ntp.org {
        }
        server 1.ubnt.pool.ntp.org {
        }
        server 2.ubnt.pool.ntp.org {
        }
        server 3.ubnt.pool.ntp.org {
        }
    }
    syslog {
        global {
            facility all {
                level notice
            }
            facility protocols {
                level debug
            }
        }
    }
    time-zone America/Chicago
}

          Hmm. I can tell you this. I had issues with my DVR initially, and I had to get them to come out and "fix" it which was factory resetting it.  I would probably suggest making your own thread and seeing if you can get responses on your issue.  

           

          Based on what I see, your internet comes in on ETH0 and is forwarded to ETH2. That is happening.  What is not happening is that the Set-Top Boxes and the DVR are not communicating with each other via multicast.

           

          I just went downstairs and tested a tv playing a recording from my dvr and it worked. Everything I've done is by trial-and-error. (mostly trying what I find via google). I've also noticed weird issues where I had to reboot the router and my smart switch.

           

          I'll say this:

          1. I have observed multicast issues internally, but this may be misleading.. The unifi discovery just doesn't find my UAP-LR(it communicates via multicast I think), so I had to ssh into the AP and go the route of setting the inform manually, really unhappy with that. I don't know how to fix this.  If I did not have a backup from my controller, I would be up a creek with a boat full of holes and no paddles.

           

          2. Their DVR can be factory reset. You lose all recordings on it.  I can provide that info, but provide no guarantee that it would even work, very last resort. Have you tried pulling the power on the DVR, waiting ~10s and then powering it back up?

           

          3. Is there any  hardware on ETH2 besides your 3 STB?

          ***

          I am only providing the knowledge of how to do this, you lose everything on your DVR, even your scheduled items!
